# 复杂场景指导

> 本文件包含 PRD 中处理离线模式、平台差异和渐进增强策略的详细参考。

## 部分离线模式

明确标注每个功能的离线要求：`必须离线` / `降级可用` / `需要网络`。

```markdown
| 功能 | 离线策略 |
|------|----------|
| 文件导入/浏览 | 必须离线 — 核心工作流 |
| 全文搜索 | 必须离线 — 使用本地 SQLite FTS5 |
| 标签同步 | 降级可用 — 离线时本地排队，联网后自动同步 |
| 云端备份 | 需要网络 — 明确提示用户需要联网 |
| 许可证验证 | 降级可用 — 本地缓存 7 天，过期后仅需联网一次 |
```

## 渐进增强

```markdown
基础体验（所有平台）：文件导入 → 浏览 → 搜索 → 标签
增强层 1（桌面端）：批量操作 → 快捷键 → 拖拽 → 系统集成
增强层 2（付费版）：云同步 → AI 标签 → 团队协作
```

## 移动端差异策略

桌面和移动端共享 Rust Core，但以下维度需要在 PRD 中明确区分：

### 功能可用性矩阵

```markdown
| 功能 | 桌面 | iOS | Android | 差异处理 |
|------|------|-----|---------|----------|
| 自动更新 | Updater 插件 | App Store 审核 | Play Store / 自分发 | 桌面用 Tauri Updater，移动端走商店审核 |
| 后台任务 | 系统 tray + 常驻 | 后台刷新（受限 30s） | WorkManager | 桌面常驻，移动端改为用户触发 + 本地队列 |
| 文件访问 | 任意路径（需 Capabilities） | 沙盒限制 | Scoped Storage | 桌面直接路径，移动端用系统 Picker |
| 推送通知 | 不需要 | APNs (P1) | FCM (P1) | 移动端必须，桌面不需要 |
| 生物识别 | 不需要 | Face ID / Touch ID (P2) | 指纹 / 面部 (P2) | 敏感操作二次验证 |
| 键盘输入 | 物理键盘 | 虚拟键盘推高 viewport | 同左 | 移动端需适配 visualViewport |
```

### 网络策略

```markdown
| 场景 | 桌面策略 | 移动端策略 |
|------|---------|-----------|
| API 调用 | 直接调用 + 超时 | 本地优先 → 后台同步 + 指数退避 |
| WebSocket | 长连接 | 前台长连接 + 后台断开，用推送唤醒 |
| 离线编辑 | 少见（网络稳定） | 必须支持 → 本地 SQLite + 冲突解决策略 |
| 大文件上传 | 直传 | 分片 + 断点续传 + 后台上传 |
```

### PRD 必须回答的移动端问题

1. V1.0 是否包含移动端？如果包含，哪些 P0 功能在移动端降级为 P1/P2？
2. 移动端的最低系统版本？（iOS 15+ / Android 8+ 是 Tauri v2 最低要求）
3. 是否需要同时上架 App Store 和 Google Play？自分发的 Android APK 是否需要？
4. 移动端的离线优先级？哪些功能必须离线可用？
5. 数据同步冲突策略？最后写入胜出 (LWW) vs 用户选择？
