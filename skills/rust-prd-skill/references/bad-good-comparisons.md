# BAD / GOOD PRD 对比

> 本文件包含常见 PRD 编写错误与正确写法的对比，帮助识别和避免模糊描述。

## 1. 需求背景：空洞 vs 量化差距分析

```markdown
// BAD — 空洞的背景描述
"我们需要一个文件管理工具，市场上这类产品不多。"

// GOOD — 量化的背景 + 明确的差距分析
"企业用户日均处理 200+ 本地文件，现有方案（Finder/Explorer）缺少批量元数据编辑和
跨平台标签同步。竞品 A 仅支持 macOS，竞品 B 需联网且月费 $15。
我们的机会：原生性能 + 离线优先 + 跨平台，零订阅费。"
```

## 2. 目标用户：泛泛而谈 vs 具体画像

```markdown
// BAD — 泛泛而谈
"所有需要管理文件的用户。"

// GOOD — 具体画像 + 场景区分
"主要用户：自由摄影师（20-45 岁，管理 10K+ 照片/视频素材）。
次要用户：小型设计团队（3-10 人，需要共享标签体系）。
桌面端场景：批量导入 + 元数据编辑 + 智能分类（高频，单次 30min+）。
移动端场景：快速浏览 + 标记 + 分享（低频，单次 < 3min）。"
```

## 3. 功能描述：模糊 vs 可测试

```markdown
// BAD — 无法验证是否完成
"支持导出功能。" "搜索要好用。" "UI 要美观。"

// GOOD — 每条都有可测试的验收标准
| 功能 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| 批量导入 | P0 | 拖拽/选择导入本地文件 | AC1: 支持 1000 文件批量导入 < 5s |
| | | | AC2: 导入中可取消，已导入部分保留 |
| | | | AC3: 重复文件检测（MD5），自动跳过 |
| 全文搜索 | P1 | 按文件名/标签/元数据搜索 | AC1: 10K 文件库中搜索响应 < 200ms |
| | | | AC2: 支持布尔表达式（AND/OR/NOT） |
| 自定义标签 | P0 | 用户创建颜色标签并批量应用 | AC1: 每文件最多 20 标签 |
| | | | AC2: 标签跨平台同步（桌面端为主） |
```

## 4. 性能目标：缺失 vs 量化

```markdown
// BAD — 没有性能要求
"应用启动要快。" "搜索不能卡。" "不能占太多内存。"

// GOOD — 具体指标 + 测量条件
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 冷启动 | < 800ms | 从双击图标到首屏可交互（Release build） |
| 热启动 | < 200ms | 从最小化恢复到可交互 |
| 内存（空闲） | < 80MB | 空闲 5 分钟后 Activity Monitor / Task Manager |
| 内存（峰值） | < 300MB | 导入 1000 文件时 |
| 安装包大小 | < 25MB (Win/Mac), < 15MB (Linux) | Release installer |
| IPC 延迟 | < 5ms P99 | 前端 → Rust 单次 invoke（本地数据） |
| 搜索响应 | < 200ms | 10K 文件库全文搜索 |
```

## 5. 技术约束：遗漏 vs 完整矩阵

```markdown
// BAD — 遗漏关键约束
"用 Rust + Tauri 开发。"

// GOOD — 完整约束矩阵
- Rust: 1.80+（MSRV 写入 CI）
- Tauri: v2.x，前端 @tauri-apps/api v2
- 前端：React 19 + TypeScript 5.x + Vite 6 + Tailwind 4
- 最低系统：Windows 10 21H2 / macOS 13 / Ubuntu 22.04 / iOS 16 / Android 13
- 离线：核心功能 100% 离线可用，同步功能需要网络
- 存储：本地 SQLite（tauri-plugin-sql），配置用 tauri-plugin-store
- 分发：Windows (NSIS + MSI), macOS (DMG + App Store), Linux (AppImage + deb)
```

## 6. 跨平台策略：忽略 vs 显式规划

```markdown
// BAD — 一句"跨平台"带过
"用 Tauri 实现跨平台。"

// GOOD — 显式标注每平台差异和优先级
"V1.0 发布 Windows + macOS（P0），Linux AppImage（P1）。
移动端 V1.x：iOS 先于 Android（iOS 用户付费意愿高）。
macOS 独有：Touch Bar 支持（P2）、Finder 集成（P1）。
Windows 独有：资源管理器右键菜单集成（P1）、任务栏进度条（P2）。
Linux 独有：XDG 目录规范（P0）、系统托盘（P1）。"
```

## 7. 安全/隐私缺失 vs 合规设计

```markdown
// BAD — 无任何数据隐私说明
"用户数据保存在本地，应用不联网。"

// GOOD — 明确合规要求 + 数据收集清单
| 数据类型 | 收集 | 存储 | 用途 | 用户可控 |
|---------|------|------|------|----------|
| 文件元数据 | 否 | 仅本地 | 核心功能 | 用户可删除 |
| 使用统计 | 是（可选） | 服务端 | 产品优化 | 设置中可关闭 |
| 崩溃日志 | 是 | 服务端 | 稳定性 | 自动脱敏 |
合规：GDPR 数据最小化、CCPA 用户删除权（7 天内响应）、App Store 5.1.1。
加密：tauri-plugin-store + OS keychain。网络：全链路 TLS 1.3。
```

## 8. 无发布策略 vs 分阶段发布

```markdown
// BAD — 模糊的发布描述
"开发完成后发布到所有平台。"

// GOOD — 分阶段发布 + 平台差异时间线
| 阶段 | 平台 | 范围 | 里程碑 |
|------|------|------|--------|
| Alpha | Win/Mac | P0 功能 | 崩溃率 < 1% |
| Beta | Win/Mac | P0+P1 | 500 用户，NPS > 40 |
| RC | Win/Mac | 候选版本 | 零 P0 bug，安全审计通过 |
| V1.0 | Win/Mac | 正式发布 | App Store / 网站分发 |
| V1.1 | +Linux | 补齐 Linux | AppImage + deb |
| V1.2 | +iOS | 移动端 | TestFlight → App Store |
回滚：每阶段保留上一版本。灰度更新：5% → 24h 无崩溃 → 全量。
```
