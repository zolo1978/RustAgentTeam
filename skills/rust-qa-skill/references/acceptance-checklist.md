# PRD 验收清单模板

## 使用说明

1. 从 PRD 提取每个功能的 Acceptance Criteria
2. 为每条 AC 填写下表
3. 每个功能至少 1 条可自动验证的检查
4. 所有 P0 检查通过后才能标记功能为"完成"

## 验收清单表格

| AC 编号 | 描述 | 验证方法 | 预期结果 | 实际结果 | 状态 |
|---------|------|---------|---------|---------|------|
| AC-1.1 | | 自动/grep/手动 | | | [ ] |
| AC-1.2 | | 自动/grep/手动 | | | [ ] |

### 验证方法说明

| 方法 | 适用场景 | 自动化程度 |
|------|---------|-----------|
| 自动 | 可通过脚本或测试代码验证 | 完全自动 |
| grep | 代码中是否存在特定模式 | 半自动 |
| 手动 | UI 交互、视觉效果、用户感知 | 人工 |

## 阻塞项

列出所有 FAIL 状态的 P0 检查项，需要开发修复后重新验证：

```
- [AC-1.2] 描述... → 失败原因: ...
- [AC-3.1] 描述... → 失败原因: ...
```

## 通过标准

- 所有 P0 AC 通过
- P1 AC 通过率 >= 80%
- 无 TODO/FIXME/stub 残留（Section 3 验证）
- IPC 契约验证通过（Section 2 验证）
- Smoke Test 全部通过（Section 6 验证）
