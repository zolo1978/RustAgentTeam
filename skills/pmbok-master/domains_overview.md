# PMBOK第七版 - 8大项目绩效域概览

## 从知识领域到绩效域的转变

**PMBOK第六版**: 10大知识领域(整合、范围、进度、成本、质量、资源、沟通、风险、采购、干系人)

**PMBOK第七版**: 8大绩效域 - 从静态知识分类到动态交互系统

### 为什么改变?

```
知识领域思维:              绩效域思维:
分类管理 ────────────────> 系统交互
独立优化 ────────────────> 整体优化
过程导向 ────────────────> 成果导向
```

## 绩效域详解

### 1. 干系人绩效域 (Stakeholder Performance Domain)

**核心问题**: 谁会影响项目?谁会被项目影响?如何有效参与?

**关键活动**:
- 识别干系人(内部/外部/支持者/反对者)
- 分析干系人(权力、利益、影响、态度)
- 制定参与策略(告知/咨询/参与/协作/授权)
- 监督参与有效性

**成功指标**:
✅ 所有关键干系人被识别和参与
✅ 干系人满意度达标
✅ 冲突得到及时解决
✅ 价值获得干系人认可

**常用工具**:
- 干系人登记册
- 权力-利益矩阵
- 参与评估矩阵
- 沟通计划

**与12原则关联**: 
- 原则③ 有效地干系人参与
- 原则① 管家式管理
- 原则④ 聚焦价值

### 2. 团队绩效域 (Team Performance Domain)

**核心问题**: 如何组建和培养高绩效团队?

**关键活动**:
- 定义团队结构(职能型/项目型/矩阵型)
- 明确角色职责(RACI)
- 团队发展(塔克曼五阶段)
- 营造协作文化
- 冲突管理和激励

**团队发展阶段**:
```
形成期(Forming) → 震荡期(Storming) → 规范期(Norming) 
    → 表现期(Performing) → 解散期(Adjourning)
```

**成功指标**:
✅ 团队士气高
✅ 协作效率强
✅ 技能持续成长
✅ 冲突建设性解决

**常用工具**:
- 团队章程
- RACI矩阵
- 技能矩阵
- 团队建设活动

**与12原则关联**:
- 原则② 营造协作环境
- 原则⑥ 展现领导力
- 原则⑦ 根据情境裁剪

---

### 3. 开发方法与生命周期绩效域 (Development Approach and Life Cycle Performance Domain)

**核心问题**: 用什么方法开发?如何安排节奏?

**方法选择光谱**:
```
预测型 ←────────── 混合型 ──────────→ 适应型
(瀑布)             (门径+敏捷)          (Scrum)

需求清晰稳定 ──────────────────────> 需求模糊多变
```

**关键决策因素**:
- 需求确定性
- 技术复杂度
- 干系人参与度
- 交付频率
- 风险容忍度

**成功指标**:
✅ 方法与项目特征匹配
✅ 交付节奏稳定
✅ 过程持续改进
✅ 团队适应良好

**常用方法**:
- 预测型: 瀑布、V模型
- 适应型: Scrum、看板、XP
- 混合型: 门径+Scrum、水-敏捷
- 增量型: IPD

**与12原则关联**:
- 原则⑦ 根据情境裁剪
- 原则⑪ 拥抱适应性
- 原则⑨ 驾驭复杂性

### 4. 规划绩效域 (Planning Performance Domain)

**核心问题**: 如何制定可执行的计划?

**规划的核心要素**:
```yaml
范围规划:
  - WBS工作分解
  - 需求管理
  - 范围基准

进度规划:
  - 活动定义
  - 网络图
  - 关键路径
  - 资源平衡

成本规划:
  - 估算方法
  - 预算基准
  - 应急储备

其他:
  - 质量计划
  - 资源计划
  - 风险计划
  - 采购计划
  - 沟通计划
```

**规划方法对比**:

| 方面 | 预测型规划 | 适应型规划 |
|------|------------|------------|
| 粒度 | 详细到任务 | 高层到史诗 |
| 时间 | 全周期一次 | 滚动式规划 |
| 基准 | 固定基准 | 灵活调整 |
| 估算 | 精确估算 | 相对估算 |

**成功指标**:
✅ 计划可执行
✅ 估算准确
✅ 变更控制有效
✅ 资源利用率高

**常用工具**:
- WBS词典
- 甘特图/网络图
- 挣值管理(EVM)
- 蒙特卡洛模拟

**与12原则关联**:
- 原则⑦ 根据情境裁剪
- 原则⑩ 优化风险应对
- 原则⑤ 系统交互

```
可交付物 ───────────────> 成果 ───────────────> 价值
(Output)                (Outcome)            (Value)

系统上线 ───────────────> 效率提升 ───────────────> ROI实现

```

**关键活动**:
- 管理需求和范围

**持续交付**:
```
持续集成(CI) → 持续交付(CD) → 持续部署
  (代码合并)     (可发布状态)    (自动上线)
```

**成功指标**:
✅ 验收通过率高

### 7. 度量绩效域 (Measurement Performance Domain)

**核心问题**: 如何度量项目绩效并支持决策?

**度量的三个层次**:
```
可交付物度量:
  - 功能点完成率
  - 缺陷密度
  - 测试覆盖率

过程度量:
  - 进度偏差(SPI)
  - 成本偏差(CPI)
  - 返工率

业务成果度量:
  - 价值实现率
  - 客户满意度
  - ROI
```

**挣值管理(EVM)**:
```
PV(计划价值) = 计划工作的预算成本
EV(挣值) = 已完成工作的预算成本
AC(实际成本) = 已完成工作的实际成本

SPI = EV / PV    (进度绩效指数)
CPI = EV / AC    (成本绩效指数)

SPI > 1.0 → 进度超前
CPI > 1.0 → 成本节约
```

**仪表盘设计原则**:
✅ 关键指标突出
✅ 可视化呈现
✅ 实时更新
✅ 钻取详情

**成功指标**:
✅ 度量体系完善
✅ 数据质量高
✅ 趋势分析准确
✅ 支持决策有效

**与12原则关联**:
- 原则④ 聚焦价值
- 原则⑤ 系统交互
- 原则⑪ 适应性

### 8. 不确定性绩效域 (Uncertainty Performance Domain)

**核心问题**: 如何应对风险和不确定性?

**不确定性的三种形式**:
```
风险(Risk): 已知的不确定性
  - 可以识别
  - 可以评估概率和影响
  - 可以制定应对计划

未知的未知(Unk-Unk): 未知的不确定性
  - 无法预见
  - 需要韧性和适应性
  - 快速响应能力

模糊性(Ambiguity): 多种解释
  - 需求不明确
  - 目标有歧义
  - 需要澄清
```

**风险管理流程**:
```
识别 → 分析 → 应对 → 监控
  ↑                       ↓
  └───────── 持续循环 ─────┘
```

**应对策略矩阵**:

| 威胁 | 策略 | 示例 |
|------|------|------|
| 高概率高影响 | 规避 | 改变计划 |
| 高概率中影响 | 减轻 | 增加测试 |
| 中概率高影响 | 转移 | 购买保险 |
| 低概率低影响 | 接受 | 预留储备 |

**储备管理**:
```
应急储备(Contingency Reserve):
  - 用于已识别风险
  - 项目经理管理

管理储备(Management Reserve):
  - 用于未知风险
  - 高层管理

总预算 = 成本基准 + 应急储备 + 管理储备
```

**成功指标**:
✅ 风险识别全面
✅ 应对及时有效
✅ 储备使用合理
✅ 未预见风险少

**与12原则关联**:
- 原则⑩ 优化风险应对
- 原则⑪ 拥抱适应性
- 原则⑨ 驾驭复杂性

## 8域的系统交互

### 交互关系图

```
干系人域 定义价值 ────────┐
                          ↓
团队域 协作执行 ──────────> 开发方法域 选择节奏
                          ↓
规划域 制定基准 ──────────> 项目工作域 执行工作
                          ↓
交付域 实现价值 ←──────────┘
         ↓
度量域 监控绩效 ←─────────> 不确定性域 管理风险
```

### 典型交互场景

**场景1: 干系人需求变更**
```
干系人域: 新需求提出
    ↓
交付域: 评估对价值的影响
    ↓
规划域: 分析对进度成本的影响
    ↓
不确定性域: 评估风险
    ↓
团队域: 讨论可行性
    ↓
度量域: 更新基准和指标
```

**场景2: 团队技能不足**
```
团队域: 识别技能差距
    ↓
项目工作域: 工作执行受阻
    ↓
度量域: 绩效指标下降
    ↓
规划域: 调整资源计划(培训/招聘/外包)
    ↓
不确定性域: 管理延期风险
```

## 总结

PMBOK第七版的8大绩效域:

✅ **整体性** - 从知识分类到系统整合
✅ **交互性** - 强调域之间的相互影响
✅ **成果导向** - 关注价值交付而非过程完成
✅ **灵活性** - 适用于各种项目管理方法

**关键要点**:
1. 8个域是一个有机整体,不能孤立看待
2. 每个域的绩效都会影响其他域
3. 项目成功需要8个域的平衡发展
4. 定期进行8域健康检查,及时调整
